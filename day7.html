Data Definition Language (DDL)

Data Definition Language (DDL) is an integral part of SQL (Structured Query Language), specifically designed for defining and managing the schema of a database. Unlike Data Manipulation Language (DML), which focuses on data operations such as insertion, updating, and deletion, DDL is concerned with the structure and organization of the database objects themselves. DDL commands are vital for creating, modifying, and removing database structures, ensuring the database is organized and ready for data manipulation.

Common DDL Commands:

  1. CREATE

    The CREATE command is used to create various database objects, including tables, indexes, views, and databases. It defines the structure of these objects and their attributes.

  2. ALTER

    The ALTER command modifies existing database objects. It can add, modify, or drop columns, as well as change other attributes of the object.

  3. DROP

    The DROP command deletes database objects permanently, removing both the structure and the data contained within the object.

  4. TRUNCATE

    The TRUNCATE command removes all records from a table but retains the table structure for future use. It is faster than the DELETE command as it does not generate individual row delete actions.

  5. RENAME

    The RENAME command changes the name of a database object, such as a table or column.

  6. COMMENT

    The COMMENT command is used to add descriptive comments to the data dictionary, enhancing the documentation of the database schema.
